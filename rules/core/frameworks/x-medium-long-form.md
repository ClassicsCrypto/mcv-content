# X Medium Long-Form Framework

- **Bucket:** medium-long-form
- **Length:** 401–800 characters, including spaces
- **Primary job:** one sharp argument, concise analysis, mini-guide
- **Use for:** detailed analysis, mini-guides, personal stories, framework breakdowns,
  tool/platform commentary
- **Do not use for:** live reminders, mood captions, proof-bank essays

## When to choose this bucket

Use medium-long-form when the post needs room for an argument but not an essay: one operator
lesson, one category correction, one practical framework, one proof cluster with 2–3 receipts, one
story with a clear consequence, or one tool/platform commentary post.

## Core shape

Use 5–8 short paragraphs or 6–10 tight lines, structured as an argument (not a small essay):

```text
1. Correction, failure, or tension
2. Real context
3. Proof point 1
4. Proof point 2
5. Implication
6. Decision rule or useful conclusion
```

This maps onto the argument spine the gate checks (claim, mechanism, verified anchor, stake,
reframe — see `rule.core.structure`).

## First-280 strategy

The first 280 characters must work as a complete post on their own (the platform truncates the
preview). Open on the tension/correction, not on a status dump (`rule.core.structure`
receipt-framing).

## Archetype fit

Best for tool/platform commentary, thesis-with-receipts, a short numbered framework, and
ship-updates that carry a practical consequence.

## Proof handling

Use 1–3 proof points max. Good proof is a named project/event/platform/feature/shipped artifact, a
specific number only when verified, or a current comparator mechanic translated into the brand's
own voice (never copied — `rule.core.claims-safety`). Every proof point must answer: *what does
this make obvious?*

## Anti-patterns (fail the draft)

- Sounds like generic founder/category wisdom (`rule.core.voice-register`).
- "Here are 3 lessons" when the story supports one (forced rule-of-three —
  `rule.core.humanizer`).
- Internal workflow turned into public content with no reader payoff.
- Fake precision; unsupported platform/tool claims.
- All variants repeat the same thesis (`rule.core.variant`).

## Writer checklist

- 401–800 characters including spaces.
- First 280 characters stand alone.
- Sharp thesis, tension, proof, stakes, useful conclusion.
- 1–3 proof points.
- The ending gives a rule, not a slogan.
- No hashtags (Twitter/X); no mid-sentence em dash.

## Example opening (brand: Acme Cosmos)

```text
Most people judge a beta by signups. That misses the point. The Acme Cosmos beta is worth
judging by how many people shipped something real before it closed.
```
